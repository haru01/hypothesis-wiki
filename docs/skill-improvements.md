# スキル改善バックログ

ドッグフーディングや運用の中で見つかった、`.claude/skills/` の改善余地を記録する。
各項目は「気づいた文脈／課題／改善案／対象スキル／状態」で書く。状態は `未対応` `対応中` `対応済み`。

## SI-001: `/grill` が状況・行動仮説を引き出せていない

- **対象スキル**: `/grill`
- **状態**: 未対応
- **気づいた文脈**: 本ツール自体への `/grill` 適用（2026-07-16）。CPFで課題仮説（H-004〜H-007）は複数起票できたが、**状況・行動仮説は H-001 の1件のみ**で、しかも内容は「記録しようとしている」という状況寄りの記述に留まり、顧客が現在とっている**具体的な行動**（CPF問いかけバンクQ3）を独立した反証可能な仮説として引き出せなかった。
- **課題**: CPFは「状況・行動仮説」と「課題仮説」の2タイプを形成すべきなのに、対話が課題（痛み）へ早く流れてしまい、行動仮説が手薄になる。行動の事実を押さえないまま課題を語ると、後続の検証（過去の事実を聞く問題インタビュー）の土台が弱くなる。
- **改善案**:
  1. `/grill` の手順2に「まず状況・行動仮説を1本確定させてから課題仮説に進む」順序を明記する。
  2. 問いかけで、課題（Q4以降）に進む前に**行動を尋ねるQ3を必ず1問挟む**（「その状況で顧客は現在どんな行動をとっているか？」を具体的な動作レベルで）。
  3. セッション終了時に「状況・行動仮説が最低1件あるか」を自己点検するチェックを入れる（なければ行動の深掘りに戻す）。
- **関連**: CPF playbook（`playbooks/cpf.md`）の「形成すべき仮説タイプ」と整合させる。

## SI-002: `/plan` はテストカードと併せてインタビュースクリプト雛形を出せるとよい

- **対象スキル**: `/plan`（および `/ingest` との接続）
- **状態**: 対応中（雛形を `templates/interview-script.md` に用意。スキル手順への組み込みは未）
- **気づいた文脈**: ACT-001（問題インタビュー）のテストカードを作った直後、「方法」欄だけでは現場で実際に話す会話に落ちておらず、別途スクリプトが要ると気づいた（2026-07-16）。
- **課題**: テストカードは検証の設計（目的・指標・成功基準）は与えるが、**現場で使える会話スクリプト**を与えない。毎回ゼロから作ると、CPFの鉄則（過去の事実を聞く／ソリューションを見せない／誘導しない）が抜け落ちやすい。
- **改善案**:
  1. `/plan` の手順5で、interview/demo 系のテストカードを作ったら `templates/interview-script.md` を基に**対象仮説を差し込んだスクリプト草稿**も併せて提示する（活動レコードに添付、または別ファイルとしてリンク）。
  2. スクリプトに「対象者ごとの〈自認〉〈実コスト〉○×表」を組み込み、`/ingest` の成功基準判定（切実な課題トップ3）に直結させる。
  3. デモ用（PSF）の雛形も将来的に用意する（`templates/demo-script.md`）。
- **関連**: `templates/interview-script.md`、テストカード `wiki/activities/ACT-001.md`、`/ingest` の学習カード記入。
