# デモ スクリプト雛形（PSF）

`/plan` で作ったテストカード（活動レコード・type: demo）の**方法**を、現場のデモ司会・観察スクリプトに落とすための雛形。
**作り込む前に**、プロト・モック・ペーパープロトを見せて／操作させて、価値の芯が当たっているかを**行動**で確かめる。
1セッション 30〜45分を想定。`〈…〉` を対象仮説・ソリューションに合わせて置き換えて使う。

> 心得の正典は [../playbooks/interviewing.md](../playbooks/interviewing.md)（確証バイアス・反証質問・発見型の選択権）。この雛形は現場文言（How）、interviewing.md は原則（Why）の分担。
>
> **姉妹雛形**: 提示物を見せて反応を"会話で"深掘りするなら [solution-interview-script.md](solution-interview-script.md)、課題そのものの実在を確かめる CPF なら [problem-interview-script.md](problem-interview-script.md)。ソリューションインタビューと併用してよい（デモで見せ、そのまま反応を聞く）。

## デモの鉄則

- **観察が主、説明は従。** デモは価値を「売り込む」場ではなく、相手の**反応と行動を観察する**場。喋りすぎない。
- **相手に操作させられるなら操作させる（タスク型）。** どこで詰まるか・どこを飛ばすかは、口頭の感想より雄弁。
- **前のめり・沈黙・質問・「いつ使える？」を記録する。** 好意的な〈発言〉は証拠にならない（interest ≠ intent）。判定は**デモ後の具体的な行動**（トライアル依頼・紹介・次アポ・稟議）で行う。
- **忖度を誘わない。** 「ここ便利ですよね？」と同意を取りにいかない。詰まっても即座に助け船を出さず、まず何をしようとするか観察する。

## 目的モード（始める前に選ぶ／正典は interviewing.md）

- **反証型** — 「刺さらない/使わない」を能動的に探す。刺さった反応を見たら**対で**「今のやり方で十分な点は？」を聞く。反証条件は成功基準に事前登録し後から変えない。
- **発見型** — 想定外の使い方・別用途・別の詰まりどころを surface する。相手が勝手に始めた操作・言及を別枠で拾う。

## 記入ヘッダー（セッションごとに埋める）

- 対象仮説: [[H-NNN]] …（テストカードの hypotheses と揃える）
- 提示物: 〈モックアップ / プロト / ペーパープロト〉（[[<PREFIX>-ACT-NNN]] のプロトタイプがあれば相対リンク）
- デモ形式: 〈こちらが操作して見せる / 相手に操作させる（タスク型） / 併用〉
- 対象者条件（スクリーニング）: 〈検証済みの課題を抱えている顧客像〉
- 実施日 / 対象者 / 所要時間 / 記録係 / 録音・録画可否:

## 0. 導入（3分）

> 今日は作りかけの〈テーマ〉をお見せします。完成品ではないので、粗いところは気にせず、**ピンとこない・分かりにくいところこそ**そのまま口に出してください。正解探しではありません。（録音・録画可否の確認）

## 1. デモ前の基線確認（5分）— いまどうしているか

- いま〈課題〉をどうやって凌いでいますか？（代替手段）それに何を・どれだけ払っていますか？

> ソリューションインタビューと併用する場合は、そちらの1章に寄せてよい（重複して聞かない）。

## 2. デモ本体（15分）— 見せる順序と観察

見せる画面／ステップを順に並べ、**各ステップで何を観察するか**を先に決めておく。

| # | 見せる画面・ステップ | このステップで検証したい仮説の点 | 観察する反応（前のめり／質問／沈黙／詰まり） |
|---|---|---|---|
| 1 | 〈例: トップ画面〉 | 〈価値提案が一目で伝わるか〉 | |
| 2 | 〈例: 主要機能の操作〉 | 〈課題のコアを解くと感じるか〉 | |
| 3 | 〈例: 結果・出力〉 | 〈代替手段より良いと感じるか〉 | |

- **タスク型なら**: 「〈具体的なゴール：例=先週の意思決定の経緯を辿ってみてください〉」と指示し、**黙って観察**する。どこで手が止まるか・どこを飛ばすか・何を探すかを記録。詰まっても最初は助けない。
- こちらが操作して見せる場合も、**節目ごとに「ここまでで何か気になりますか？」**と止めて反応を拾う（一気に流さない）。

## 3. デモ直後の反応（10分）

- 今のを見て（触って）、**一番刺さったのはどこ／一番いらないのはどこ**ですか？ 具体的に。
- （反証型）**逆に**、これがあっても今のやり方で十分・使わないと思う点は？
- いまの〈代替手段〉と比べてどうですか？ 乗り換えるとしたら決め手は？

## 4. 次の行動の観察（5分）— 支持の本命シグナル

**「使いたい？」の未来意向でなく、いま取れる行動を見る。**

- もし使えるなら、**次に何をしますか**？（トライアル申込／チーム共有／稟議／紹介）——その場でどこまで動けるか観察。
- 同じような方を紹介いただけますか？（前のめりの〈行動〉シグナル）

## 禁止・注意

- **売り込まない・褒め言葉を引き出しにいかない。** デモ中に価値を語り足して忖度を誘わない。
- 詰まった相手をすぐ助けない（まず観察）。誘導質問・未来仮定を成功基準の根拠にしない。
- 判定は好意的な感想でなく、**詰まりどころ**と**デモ後の具体的行動**（〈行動〉〈支払い〉）で行う。

## 記録のコツ（`/ingest` に効く）

- **観察した事実（操作・詰まり・発言）は原文/具体で記録**し、**解釈と必ず分ける**。録画があればタイムスタンプを残す。
- 各ステップの詰まり・スキップ・驚きを別枠でマーク（学習カードの「驚き・想定外」に直結）。
- 各対象者について「価値を自分の言葉で意味づけたか」「デモ後に具体的な次の行動を取ったか」を残すと、成功基準を機械的に判定できる。
