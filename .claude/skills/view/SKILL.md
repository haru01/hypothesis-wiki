# /view — ビュー生成

仮説レコード（SSoT）からビューを生成し `wiki/views/` に出力する。**ビューは生成物**であり、記録の修正はレコード側で行いビューは再生成する。

## サブコマンド

引数に応じて生成するビューを切り替える。指定がなければどれを生成するか尋ねる。

### `vp` — バリュープロポジションビュー

顧客プロファイル × 価値マップの対応を表示する。

- **顧客プロファイル**（課題側）: 状況・行動仮説、課題仮説から「顧客・ジョブ・ペイン」を並べる。
- **価値マップ**（解決側）: ソリューション仮説、買ってもらえる仮説から「提供価値・ペインリリーバー」を並べる。
- 両者の対応（どの価値がどの課題に効くか）を表で突き合わせ、対応の空白（未カバーの課題／裏付けのない価値）を明示する。
- 出力: `wiki/views/value-proposition.md`

### `list` — 全仮説テーブル

全仮説を1枚のテーブルにする。確信度でソートし、「次に検証すべき仮説」（重要度高×確信度低×未検証/検証中）を先頭に集める。

- **関連リンク列を必須にする** — テーブルに「ID・タイトル・タイプ・確信度・ステータス・ステージ・重要度」に加え、**「派生元」列**と**「関連リンク（解決／派生先／検証活動）」列**を含める（`[[H-NNN]]`・`[[ACT-NNN]]` で記述）。スカラー項目だけにせず、仮説どうしのつながり（系譜・課題↔解決策）が表だけで辿れるようにする。
- **先頭にバリューチェーン図を出す** — ビューの冒頭に **mermaid** で「繰り返す行動 → 切実な課題 → 解決策 → 買ってもらえる」の関連図を描き、ステージを貫く筋を一望できるようにする（実例: `wiki/views/hypotheses-list.md`）。
- 出力: `wiki/views/hypotheses-list.md`

### `board` — ジャベリンボード風（オプション）

実験の変遷を、仮説→活動→学習→次アクションの流れで時系列に並べる。`wiki/log.md` と ACT レコードから構成する。各項目に仮説↔活動↔意思決定の関連リンク（`[[H-NNN]]`・`[[ACT-NNN]]`・`[[DEC-NNN]]`）を持たせる。

- 出力: `wiki/views/board.md`

## 守ること

- 生成ファイルの冒頭に必ず `<!-- 生成物。手編集禁止。/view で再生成する。 -->` を入れる。
- 記録とビューが矛盾する場合、**レコードが正**。ビューを直すのではなくレコードを直して再生成する。
- ビュー生成では確信度・ステータス・log を変更しない（読み取り専用の集計）。
