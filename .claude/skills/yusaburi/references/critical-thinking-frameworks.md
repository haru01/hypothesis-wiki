# 論理的誤謬とクリティカルシンキング

思考の欠陥を特定するための、論理的誤謬とクリティカルシンキングのフレームワーク集。

## 論理的誤謬（Logical Fallacies）

### 因果関係の誤謬

1. **相関と因果の混同（Correlation ≠ Causation）**
   - 「A と B が同時に起こる」から「A が B の原因」と結論づける
   - 例：「成功者は早起きだから、早起きすれば成功する」
   - 突き方：「逆因果、第三の要因、偶然の可能性は？」

2. **後件肯定の誤謬（Affirming the Consequent）**
   - 「A ならば B」「B である」から「A である」と結論づける
   - 例：「良い製品なら売れる。売れている。だから良い製品だ」
   - 突き方：「売れる理由は他にないか？ マーケティング、タイミング、運は？」

3. **ポストホック（Post Hoc Ergo Propter Hoc）**
   - 「A の後に B が起きた」から「A が B の原因」と結論づける
   - 例：「新しい施策の後に売上が上がったから、施策が効いた」
   - 突き方：「同時期の他の要因、季節性、トレンドは考慮したか？」

### 一般化の誤謬

4. **性急な一般化（Hasty Generalization）**
   - 限られたサンプルから全体を断定
   - 例：「3社に断られたから、この市場には需要がない」
   - 突き方：「サンプルサイズは？ 選択バイアスは？ 統計的有意性は？」

5. **構成の誤謬（Fallacy of Composition）**
   - 部分が真だから全体も真と考える
   - 例：「各部署が最適化すれば、会社全体も最適化される」
   - 突き方：「部分最適と全体最適の衝突は？ 創発的な問題は？」

6. **分割の誤謬（Fallacy of Division）**
   - 全体が真だから部分も真と考える
   - 例：「この会社は儲かっているから、この事業も儲かっているはず」
   - 突き方：「内訳を見たか？ セグメント別の収益性は？」

### 前提の誤謬

7. **誤った二分法（False Dichotomy）**
   - 2つの選択肢しかないと思い込む
   - 例：「スピード重視か品質重視か選ばないと」
   - 突き方：「第三、第四の選択肢は？ トレードオフは本当に避けられないか？」

8. **循環論法（Circular Reasoning）**
   - 証明したいことを前提に使う
   - 例：「この方法が最善なのは、他の方法より優れているから」
   - 突き方：「『最善』『優れている』の定義は？ 独立した根拠は？」

9. **論点先取（Begging the Question）**
   - 証明すべきことを前提に含める
   - 例：「この画期的なアイデアを実行すべきだ」（画期的であることが未証明）
   - 突き方：「画期的である根拠は？ 誰がどう判断した？」

### 権威と感情の誤謬

10. **権威への訴え（Appeal to Authority）**
    - 専門外の権威や、権威の肩書だけで論拠とする
    - 例：「有名な経営者が言っているから正しい」
    - 突き方：「その人の専門分野は？ 利益相反は？ エビデンスは？」

11. **多数論証（Bandwagon Fallacy）**
    - 「皆がやっているから正しい」
    - 例：「競合も導入しているから、うちも導入すべき」
    - 突き方：「競合の前提条件とあなたの前提条件は同じか？」

12. **感情への訴え（Appeal to Emotion）**
    - 恐怖、憐憫、興奮などで判断を歪める
    - 例：「このチャンスを逃したら、もう二度と来ません！」
    - 突き方：「感情を抜いた冷静な損益分析は？」

## クリティカルシンキングのチェックリスト

### 前提の吟味
- [ ] 暗黙の前提を明示化したか？
- [ ] その前提は常に成り立つか？
- [ ] 前提が崩れた時のプランBはあるか？

### 証拠の検証
- [ ] 主張を支える証拠は何か？
- [ ] 証拠の質（サンプルサイズ、バイアス、信頼性）は？
- [ ] 反証を探したか？

### 代替案の探索
- [ ] 他の解釈、他の選択肢を考えたか？
- [ ] 「当たり前」を疑ったか？
- [ ] 異なる視点からの見え方は？

### 一貫性の検証
- [ ] 論理の飛躍はないか？
- [ ] 矛盾する主張をしていないか？
- [ ] スケール、時間軸、文脈を混同していないか？

### 利害関係の確認
- [ ] 誰が得をするか？
- [ ] 情報源の利益相反は？
- [ ] 自分の願望が判断を歪めていないか？

## ソクラテス式問答法

### 明確化の問い
- 「それはどういう意味ですか？」
- 「具体例を教えてください」
- 「別の言葉で言い換えると？」

### 前提を問う問い
- 「それを前提としているのはなぜですか？」
- 「その前提が間違っていたらどうなりますか？」
- 「他の前提は考えられませんか？」

### 根拠を問う問い
- 「それはどうやってわかりますか？」
- 「証拠はありますか？」
- 「その証拠はどれくらい信頼できますか？」

### 視点を問う問い
- 「他の人はどう見るでしょうか？」
- 「反対の立場から見ると？」
- 「10年後から見ると？」

### 影響を問う問い
- 「それが実現したら何が起こりますか？」
- 「意図しない結果は？」
- 「次の一手は？ そのまた次は？」

### メタ認知の問い
- 「なぜ今、その質問をしているのですか？」
- 「あなたの思考プロセスは？」
- 「何があなたの判断を歪めている可能性がありますか？」

## デバッグ思考フレームワーク

ソフトウェアのデバッグ手法を思考に応用：

1. **再現性の確認**：その思考パターンは繰り返し現れるか？
2. **最小再現ケース**：問題の本質だけを抜き出すと？
3. **境界値テスト**：極端な場合でも成り立つか？
4. **仮説検証**：原因の仮説を立て、反証を探す
5. **ステップ実行**：推論を一行ずつ検証する
6. **状態の確認**：今、どんな前提を持っているか？

## レッドチーム思考

意図的に攻撃者になる：

1. **脆弱性診断**：この計画の最大の弱点は？
2. **攻撃シナリオ**：悪意ある第三者ならどう妨害する？
3. **最悪ケース分析**：何が起きれば致命傷か？
4. **前提破壊**：1つの前提が崩れたら連鎖崩壊するか？
5. **タイミング攻撃**：最悪のタイミングで問題が起きたら？

## 使用ガイドライン

- このリストを**チェックリスト的に機械適用しない**
- ユーザーの思考に現れた具体的な誤謬を**1〜2個**に絞って指摘
- 誤謬の名前より、**なぜ危険か**を説明する
- 批判だけでなく、**どう直すか**も示す
