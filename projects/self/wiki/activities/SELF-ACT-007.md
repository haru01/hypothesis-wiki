# 強制モード vs 助言モードの自己A/Bで H-016（記録一体の強制力）を検証

対象仮説: [[SELF-H-016]]

<!-- H-016 は「強制型 > 助言型」という比較仮説。外部被験者がいないため、同一実践者で2モードを切り替える自己内A/Bで比較の腕を作る。 -->

## テストカード（検証前に記入・後から書き換えない）

- **目的**: [[SELF-H-016]]「記録と一体化した不変ルールの**強制力**が、助言型AI批判より確信度規律を守らせる」を、自分で2モードを切り替えて確かめる。①**強制を回避しないか**（反証条件①） ②**強制モードのほうが助言モードより進路変更が多いか**（反証条件②）。
- **方法**: demo（自己ドッグフーディングの自己内A/B。外部被験者を伴わないため記録シートで代替＝ [[SELF-ACT-005]] [[SELF-ACT-003]] と同型）。実際のスキル運用（`/refine` `/plan` `/ingest` `/decide`）中に、AIが interest≠intent・根拠なき確信度上げを検知した場面で、2条件を意図的に切り替える:
  - **強制モード**: 根拠リンクがなければ確信度を上げさせない不変ルールを厳格適用し、指摘を確信度履歴・巻き戻し点と一体で残す。
  - **助言モード**: AIは「取り違えかもしれない」と**助言だけ**して強制しない（実践者は無視できる）。
  - 各指摘イベントを下の記録シートに1行記録し、直後の行動（進路変更の有無）と、強制モードでの回避の試みを記す。

  ### 記録シート（イベントごとに1行）

  | # | モード（強制/助言） | 場面（どのスキル・何の確信度/判断か） | 進路変更したか（○=保留/反証追加/根拠補完 / ×=変えず） | 強制モードで回避を試みたか（○=抜け道/根拠なしで上げたがった / ×=受け入れ / —=助言モード） |
  |---|---|---|---|---|
  | 1 | | | | |
  | 2 | | | | |

- **指標**:
  1. 強制モード: 指摘数／うち進路変更数／うち回避を試みた数
  2. 助言モード: 指摘数／うち進路変更数
  3. 両モードの**進路変更率の差**（強制 − 助言）

- **成功基準**（開始後は書き換えない）:
  - (a) 強制モードで**回避が少数**にとどまり、**過半で進路変更**が起きる（＝反証条件①を回避）。
  - (b) 強制モードの進路変更率が助言モードより**明確に高い**（目安: +2件以上または率で明らかな差。＝反証条件②を回避、強制力の優位）。
  - (a)(b) 両方で支持寄り／(a) のみで判断保留／(a) 不成立で反証。
  - **正直な限界**: 実践者が条件も仮説も知っているため**反応性バイアスが大きい**（n=1・自己奉仕バイアス・条件の意図的操作）。弱い証拠として扱い、`sources/`・ビューに「**実在被験者未検証**」を明示し、支持でも確信度を上げすぎない（成功基準未達なら原則上げない＝ SI-008 の3値判定に従う）。
