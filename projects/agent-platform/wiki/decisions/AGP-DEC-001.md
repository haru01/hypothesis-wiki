# CPF継続（persevere）— 実データでの問題インタビュー再実施を最優先

現ステージ **CPF** を継続する。FPF への移行はしない。

## 達成度チェック（移行基準に照らす）

[playbooks/cpf.md](../../../../playbooks/cpf.md) の FPF 移行基準と現状:

| 移行基準 | 現状 | 判定 |
|---|---|---|
| 複数顧客（5人以上）が同一課題を自認 | 架空6名中3名（[[AGP-ACT-002]]）。実データは0名 | ✗（架空のみ） |
| 課題仮説の確信度7以上・対応する行動仮説7以上 | [[AGP-H-003]]=6・[[AGP-H-001]]=6（いずれも架空由来・上限8） | ✗ |
| 代替手段に実コストを払う証拠 | 架空では有（半日〜2日）。実観測はなし | ✗（実証拠なし） |

→ **移行基準は未達**。根拠が架空シミュレーションで、実データ未検証のため確信度は上限8で頭打ち。

## 選択肢と判断理由

- **A. FPFへ移行** — 却下。移行基準未達、かつ根拠が架空。実顧客の実在を確かめずに進むと砂上の楼閣。
- **B. ピボット** — 却下。まだ核心課題が反証されたわけではない（架空では支持）。方向転換の根拠がない。
- **C. persevere（採用）** — CPFを継続し、[[AGP-ACT-002]] と同じ設計で**実データの問題インタビュー**を回す。特に [[AGP-H-007]]（セグメント依存）を確かめるため、監査・再作成が発生する役割と、そうでない役割を分けて対象化する。

## 巻き戻しポイント

- **誤りのシグナル**: 実データの問題インタビューで、〈自認〉かつ〈実コスト〉が5名中3名に満たない／反証シグナル（P-B型「既存運用で辿れる」）が過半を占める。
- **戻り先**: [[AGP-H-003]] を確信度3-4（二次情報水準）へ引き戻し、status を未検証に戻す。核心クラスタ（[[AGP-H-001]]→[[AGP-H-003]]）の課題設定自体を [[AGP-H-007]] のセグメント観点で問い直す（＝この persevere 判断の前提に戻る）。

## 次の一手（戦略的現在地）

**実データでのCPF検証が現在地**。架空で立てた核心チェーンを、実顧客で「切実な課題トップは何か・誰にとってか」を確かめに行く。
